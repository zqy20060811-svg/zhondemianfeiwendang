### 3. MySQL 建表SQL

```sql
CREATE DATABASE IF NOT EXISTS zhongdemianfeiwendang
    CHARACTER SET utf8mb4
    COLLATE utf8mb4_unicode_ci;

USE zhongdemianfeiwendang;

-- 用户表
CREATE TABLE sys_user (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    username VARCHAR(50) NOT NULL COMMENT '用户名',
    password VARCHAR(100) NOT NULL COMMENT '加密密码',
    email VARCHAR(100) NOT NULL COMMENT '邮箱',
    avatar VARCHAR(255) DEFAULT NULL COMMENT '头像URL',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态 0禁用 1启用',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    UNIQUE KEY uk_username (username),
    UNIQUE KEY uk_email (email)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='用户表';

-- 文档信息表
CREATE TABLE doc_info (
    id VARCHAR(32) NOT NULL PRIMARY KEY COMMENT '文档ID',
    title VARCHAR(200) NOT NULL DEFAULT '未命名文档' COMMENT '文档标题',
    content LONGTEXT DEFAULT NULL COMMENT '文档内容',
    owner_id BIGINT NOT NULL COMMENT '创建人ID',
    version INT NOT NULL DEFAULT 1 COMMENT '当前版本号',
    status TINYINT NOT NULL DEFAULT 1 COMMENT '状态 0删除 1正常',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    update_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间',
    INDEX idx_owner_id (owner_id),
    INDEX idx_status (status)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档信息表';

-- 文档协作者表
CREATE TABLE doc_collaborator (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    doc_id VARCHAR(32) NOT NULL COMMENT '文档ID',
    user_id BIGINT NOT NULL COMMENT '用户ID',
    permission VARCHAR(10) NOT NULL DEFAULT 'VIEW' COMMENT '权限 VIEW/EDIT',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    UNIQUE KEY uk_doc_user (doc_id, user_id),
    INDEX idx_doc_id (doc_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档协作者表';

-- 文档分享表
CREATE TABLE doc_share (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    doc_id VARCHAR(32) NOT NULL COMMENT '文档ID',
    share_code VARCHAR(32) NOT NULL COMMENT '分享码',
    permission VARCHAR(10) NOT NULL DEFAULT 'VIEW' COMMENT '默认权限',
    expire_time DATETIME DEFAULT NULL COMMENT '过期时间',
    create_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '创建时间',
    UNIQUE KEY uk_share_code (share_code),
    INDEX idx_doc_id (doc_id)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='文档分享表';

-- 回收站表
CREATE TABLE recycle_bin (
    id BIGINT AUTO_INCREMENT PRIMARY KEY COMMENT '主键ID',
    doc_id VARCHAR(32) NOT NULL COMMENT '文档ID',
    title VARCHAR(200) NOT NULL COMMENT '文档标题',
    owner_id BIGINT NOT NULL COMMENT '原拥有者ID',
    delete_time DATETIME NOT NULL DEFAULT CURRENT_TIMESTAMP COMMENT '删除时间',
    expire_time DATETIME NOT NULL COMMENT '彻底删除时间',
    INDEX idx_owner_id (owner_id),
    INDEX idx_expire_time (expire_time)
) ENGINE=InnoDB DEFAULT CHARSET=utf8mb4 COMMENT='回收站表';
```

## 二、Redis 数据设计

### 1. Redis 连接信息

- **Host**: `localhost`
- **Port**: `6379`
- **Password**: `1234`
- **Database**: `0`

### 2. Key 设计规范

| Key 模式                              | 类型       | 说明                  | 过期时间 |
| ------------------------------------- | ---------- | --------------------- | -------- |
| `user:token:{token}`                  | String     | JWT Token 映射用户ID  | 7天      |
| `doc:content:{docId}`                 | String     | 文档实时内容缓存      | -        |
| `doc:version:{docId}`                 | String     | 文档当前版本号        | -        |
| `doc:history:{docId}`                 | List       | 文档操作历史(OT)      | -        |
| `doc:versions:{docId}`                | Hash       | 文档各版本内容        | -        |
| `doc:online:{docId}`                  | Hash       | 在线协作者信息        | -        |
| `doc:cursor:{docId}`                  | Hash       | 用户光标位置          | -        |
| `doc:lock:{docId}`                    | String     | 文档编辑锁            | 30秒     |
| `user:docs:{userId}`                  | Set        | 用户文档列表          | -        |
| `share:code:{shareCode}`              | String     | 分享码映射文档ID      | 7天      |

### 3. 数据结构详解

#### 3.1 文档实时内容缓存

```
Key: doc:content:{docId}
Type: String
Value: "# 1. 项目背景\n随着团队规模的扩大..."
```

#### 3.2 文档版本号

```
Key: doc:version:{docId}
Type: String
Value: "5"
```

#### 3.3 文档操作历史 (OT算法用)

```
Key: doc:history:{docId}
Type: List
Value: [
  '{"type":"INSERT","position":100,"content":"文本","version":1,"userId":1,"timestamp":1710900000}',
  '{"type":"DELETE","position":105,"length":3,"version":2,"userId":2,"timestamp":1710900100}'
]
```

#### 3.4 文档各版本内容

```
Key: doc:versions:{docId}
Type: Hash
Field: "1" -> Value: "{\"content\":\"...\",\"editorId\":1,\"editTime\":\"...\"}"
Field: "2" -> Value: "{\"content\":\"...\",\"editorId\":2,\"editTime\":\"...\"}"
Field: "3" -> Value: "{\"content\":\"...\",\"editorId\":1,\"editTime\":\"...\"}"
```

#### 3.5 在线协作者

```
Key: doc:online:{docId}
Type: Hash
Field: "1" -> Value: "{\"username\":\"张三\",\"avatar\":\"...\",\"status\":\"editing\",\"joinTime\":1710900000}"
Field: "2" -> Value: "{\"username\":\"李四\",\"avatar\":\"...\",\"status\":\"viewing\",\"joinTime\":1710900100}"
```

#### 3.6 用户光标位置

```
Key: doc:cursor:{docId}
Type: Hash
Field: "1" -> Value: "{\"position\":150,\"color\":\"#4A90D9\"}"
Field: "2" -> Value: "{\"position\":200,\"color\":\"#FF6B6B\"}"
```

#### 3.7 文档编辑锁

```
Key: doc:lock:{docId}
Type: String
Value: "1"  (用户ID)
TTL: 30秒
```

#### 3.8 用户文档列表

```
Key: user:docs:{userId}
Type: Set
Value: ["doc_123456", "doc_789012", "doc_345678"]
```

#### 3.9 分享码映射

```
Key: share:code:{shareCode}
Type: String
Value: "doc_123456"
TTL: 7天
```

## 三、数据流转说明

### 1. 文档创建流程

1. 前端调用 `POST /doc/create`
2. 后端生成 `docId`，写入 `doc_info` 表
3. 初始化 Redis：`doc:content:{docId}` = "", `doc:version:{docId}` = "1"
4. 将 `docId` 加入 `user:docs:{userId}` Set

### 2. 协同编辑流程

1. 用户连接 WebSocket `ws://localhost:8081/ws/doc/{docId}`
2. 后端将用户信息写入 `doc:online:{docId}`
3. 用户发送编辑操作 -> 服务端 OT 转换 -> 广播给所有在线用户
4. 编辑操作追加到 `doc:history:{docId}` List
5. 定时（或手动）保存：Redis 内容 -> MySQL `doc_info.content`

### 3. 版本控制流程

1. 每次保存时，`doc:version:{docId}` +1
2. 将当前内容写入 `doc:versions:{docId}` Hash (field=版本号)
3. 同时写入 MySQL `doc_info.version`
4. 获取历史版本时，优先从 Redis 读取，不存在则从 MySQL 回溯

### 4. 回收站流程

1. 删除文档：将 `doc_info.status` 改为 0，数据移入 `recycle_bin` 表
2. 恢复文档：`recycle_bin` 数据移回 `doc_info`，status 改为 1
3. 定时任务：清理 `expire_time` 已过的回收站数据
